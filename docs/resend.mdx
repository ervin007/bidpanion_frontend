# Resend Integration

This document explains how the Resend email service is integrated for sending transactional emails.

## Configuration

1.  **Provider Selection**: To use Resend, the `NEXT_PUBLIC_EMAIL_PROVIDER` environment variable must be set to `"resend"`. This can be done via the configuration UI (`/configure`) or directly in the `.env` file.

2.  **API Key**: The `RESEND_API_KEY` environment variable must contain a valid API key obtained from your [Resend account](https://resend.com).

## Sending Emails

1.  **Email Router (`src/server/email/send-email.ts`)**: This central function checks the `NEXT_PUBLIC_EMAIL_PROVIDER`. If it's "Resend", it proceeds to use the Resend-specific logic.

2.  **Resend Logic (`src/server/email/send-email-resend.ts`)**:
    - Uses the official `resend` SDK.
    - Checks if `RESEND_API_KEY` is configured (`isResendConfigured`).
    - Takes a payload (`EmailPayload`) containing `to`, `subject`, `from`, and either `react` (a React Email component) or `text`.
    - Renders React components to HTML using `@react-email/render`.
    - Handles both single recipient sends (`resend.emails.send`) and batch sends (`resend.batch.send`) if `to` is an array.
    - Includes error handling and logging for API responses.

## Usage Examples

Resend is used by the core `sendEmail` function, which is called in various parts of the application, such as:

- Sending email verification links (`src/server/auth/index.ts` within `emailVerificationConfig`).
- Sending password reset links (`src/server/auth/index.ts` within `emailAndPassword.sendResetPassword`).
- Sending email change confirmations (if enabled in `src/server/auth/index.ts`).

## Pricing and Limits

Resend offers a generous free tier, but it has limitations to be aware of:

- **Daily Sending Limit**: The **Free plan allows sending up to 100 emails per day**.
- **Domains**: The Free plan allows 1 custom sending domain.
- **Data Retention**: Email logs are retained for 1 day on the Free plan.

Reference: [Resend Pricing](https://resend.com/pricing)

**Important Considerations:**

- If you anticipate high user growth or need to send more than 100 emails daily (e.g., for email verification upon signup), the free tier limit might be reached quickly.
- **Mitigation Strategies**:
  - **Disable Email Verification**: In `src/env.ts` (or via the config UI), set `NEXT_PUBLIC_AUTH_ENABLE_EMAIL_VERIFICATION` to `false`. This removes the need to send a verification email on signup.
  - **Rely on Social Logins**: Encourage or enforce login via social providers (like GitHub, if configured) which typically don't require separate email verification through your app.
  - **Upgrade Resend Plan**: Consider upgrading to a paid Resend plan (Pro or Scale) for higher or unlimited sending limits.
